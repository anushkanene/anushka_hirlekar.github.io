Overview

SFPL.org — an information architecture case study. We are improving the digital structure of the San Francisco Public Library website to enhance content organisation, navigation, and overall findability to better serve a diverse range of library users.

Problem

The existing SFPL website faced several usability issues:

  • Complex Navigation: Users struggled to find information about booking locations, tracking and changing reservations
  • Inconsistent Information Architecture: Link names, categories and locations within the website made the content hard to find
  • Limited Content Discoverability: Important locations and programs were buried under multiple menus, reducing visibility

Redesigning the SFPL Digital Experience

Improving navigation, content discoverability, and information architecture so every library user can find what they need.

Solution

To address navigation challenges and improve content discoverability on the SFPL website, we redesigned its information architecture using a tree-testing approach.

We conducted card sorting to understand how users mentally categorised content, and created more intuitive structures based on their input. This was followed by tree testing, which confirmed significant improvements in task success rates. The refined IA features clearer labels, logical category groupings, and streamlined pathways to help users complete their tasks with ease.

Additionally the solution included a revised site map and a clickable prototype showcasing the improved structure — enabling all types of library patrons to explore, engage, and find the resources they need.

03 Gap Analysis & Tree Testing

Gap Analysis & Tree Testing

We conducted a gap analysis, by mapping the user tasks to the information architecture on the website where the existing gaps failed to meet user needs.

We also ran tree testing to identify "what did not determine" (or what left current SFPL users confused).

Key Findings from Tree Testing:

  • Lower Success Rate: Tasks 2, 3, 5 and 9 had the lowest success rates (53%), reporting poor findability of key resources
  • Unclear Labelling: Users frequently click unexpected labels, highlighting issues with the IA
  • Navigation Inefficiencies: The structure made it difficult to discover digital resources and find relevant information

05 Tree Testing & Success Rate Improvement

Tree Testing & Success Rate Improvement

We repeated the tree testing, conducting another round of tree testing using OptimalSort.

Key Results

  • Overall Success Rate Improvement: Tasks previously failing were now succeeding at a much higher rate, making the IA far more intuitive to users
  • Drop in Task 5 Success: Task 5's success rate dropped from 100% to 75% — outcome indicated this key step needed extra alignment
  • Sublime Refinement Required: This results highlighted issues for further system advancement — this addressed adjustments to reduce task failures
